# Автоматизированный тест Playwright для страницы eventswidget

Этот репозиторий содержит автоматизированный тест (Playwright + TypeScript) для веб-страницы:

```
https://dev.3snet.info/eventswidget/
```

Цель — проверить базовую работоспособность виджета событий и наличие ключевых элементов интерфейса.

## Что проверяет тест

### Загрузка страницы
- Успешный возврат HTTP-кода 200.
- Корректная отрисовка DOM после загрузки.

### Наличие основного контейнера виджета
Тест ищет разные варианты селекторов (`#events-widget`, `.events-widget`, `.event-list` и др.).
Если точный селектор отсутствует, используется fallback-логика.

### Наличие списка событий
- Поиск карточек событий по нескольким вариантам селекторов.
- Проверка, что отображается хотя бы один элемент.

### Проверка структуры первой карточки события
- Наличие заголовка или похожего текстового блока.
- Наличие даты/времени (если отображается).
- Проверка, что текстовые поля не пустые.

### Проверка взаимодействия («Подробнее»)
Если у события есть кнопка или ссылка «Подробнее» / «Show more» / «Details»:
- Выполняется клик.
- Проверяется отображение модального окна или расширенной секции.

## Быстрый запуск

После клонирования репозитория выполните:

### 1. Установка зависимостей
```bash
npm ci
```

### 2. Запуск теста
```bash
npm run test:ci
```

### Запуск с интерфейсом браузера
```bash
npx playwright test --headed
```

## Особенности и наблюдения

- Страница может изменяться — текст кнопок, структура карточек, классы.
  Поэтому селекторы подобраны максимально гибко.
- Если виджет загружается медленно, увеличьте timeout в тесте.
- Интерфейс виджета не стандартизирован — часть проверок является эвристической.
